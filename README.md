# Paradox Localisation Helper

> **Навайбкожено с Kimi K2.5, но сильно отредактироваонно**

Инструмент для работы с файлами локализации модов к играм Paradox Interactive (Stellaris, Crusader Kings, Europa Universalis, Hearts of Iron и др.; тестировалось на моде на Stellaris).

## Возможности

- **Единый отчёт** — полный анализ: изменения версий, прогресс перевода, дубликаты, лишние ключи
- **Операции над множествами** — удобная работа с ключами через Except, Modified, Union, Intersect
- **Генерация YML** — автоматическое создание файлов для оставшейся работы
- **Слияние переводов** — объединение устаревшего перевода с обновлением
- **Сохранение настроек** — автоматическое сохранение путей и языков в конфиге

## Быстрый старт

### Требования
- .NET 10 SDK

### Установка
```bash
git clone https://github.com/username/ParadoxLocalisationHelper.git
cd ParadoxLocalisationHelper
dotnet build
dotnet run
```

### Использование

Запустите программу без аргументов:

```bash
dotnet run
# или
ParadoxLocalisationHelper.exe
```

Откроется интерактивное меню:

```
=== Paradox Localisation Helper ===
      Помощник локализации Paradox

Текущие настройки:
  [1] Новая версия (оригинал):     (не задан)
  [2] Старая версия (оригинал):    (не задан)
  [3] Устаревший перевод:          (не задан)
  [4] Обновление перевода:         (не задан)
  [5] Язык оригинала:              l_english
  [6] Язык перевода:               l_russian

Действия:
  [7] Сгенерировать отчёт
       (анализ изменений и прогресса перевода, с YML для оставшейся работы)

 [8] Объединить переводы
       (слияние устаревшего перевода с обновлением)

  [0] Выход
```

### Типичный workflow

#### Анализ изменений между версиями

```
1. Установите путь к новой версии (пункт 1)
2. Установите путь к старой версии (пункт 2)
3. Сгенерируйте отчёт (пункт 7)
→ Увидите все добавленные/удалённые/изменённые ключи
```

#### Проверка прогресса перевода

```
1. Установите путь к новой версии (пункт 1)
2. Установите путь к устаревшему переводу (пункт 3)
3. Установите путь к обновлению перевода (пункт 4)
4. Сгенерируйте отчёт (пункт 7)
→ Увидите что сделано, что осталось, дубликаты, лишние ключи
```

#### Полный цикл обновления перевода

```
1. Установите все 4 пути (пункты 1-4)
2. Сгенерируйте отчёт (пункт 7)
3. Получите YML-файлы с оставшейся работой в папке remaining_work_*
4. Переведите ключи из этих файлов
5. Выполните слияние (пункт 10)
```

## Особенности

### LocalizationStorage — удобная работа с множествами

```csharp
LocalizationStorage workToDo = addedKeys
    .Union(modifiedKeys)
    .Union(missingInOldTranslation)
    .SetName("Work To Do")
    .Describe();  // выведет в консоль и вернёт this
```

**Доступные операции:**
- `Except(other)` — ключи, которые есть в this, но нет в other
- `Modified(other)` — ключи с разными значениями
- `Unchanged(other)` — ключи с одинаковыми значениями
- `Union(other)` — объединение
- `Intersect(other)` — пересечение

### Единый отчёт

Вместо нескольких разных отчётов — один комплексный:
- Изменения между версиями (added/removed/modified)
- Работа по переводу (work to do / done / remaining)
- Дубликаты в переводах
- Лишние ключи (unnecessary)
- Одна большая таблица по всем файлам со строкой **Total**

### Генерация YML

Автоматическое создание файлов для оставшейся работы с сохранением структуры оригинала.
